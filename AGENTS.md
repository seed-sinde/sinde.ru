# Codex Notes

## Scope

- Рабочее окружение: `/home/dc/storage`.
- Связанные репозитории:
  - `sinde.ru` — публичный основной репозиторий для разработки и локального запуска.
  - `sinde.ru-ops` — приватный репозиторий с CI/CD, деплоем, серверными скриптами и эксплуатационной документацией.
- Если задача затрагивает инфраструктуру, деплой, CI/CD, runtime-конфигурацию, dev/prod расхождения или интеграцию сервисов, учитывать оба репозитория даже при `cwd` внутри одного.
- Если пользователь говорит "корень проекта", различать:
  - корень текущего репозитория;
  - корень рабочего окружения `/home/dc/storage`.
- Явно указывать, в каком репозитории выполняются действия.
- В `sinde.ru` не добавлять секреты, ключи, токены и production credentials.
- Содержимое `sinde.ru-ops` считать приватным и потенциально чувствительным.

## Working Rules

- Вносить только целевые, минимальные изменения.
- Не переписывать файл целиком без необходимости.
- Не дублировать существующую логику и типы.
- Предпочитать упрощение структуры новой абстракции.
- Удалять лишние wrapper-контейнеры.
- Если у контейнера один ребёнок и он не нужен для layout/position/semantic-role, переносить классы в ребёнка.
- Дизайн должен быть минималистичным, но контрастным.
- Не использовать границы, скругления и отдельные цвета фона у внутренних потомков контейнера, если без этого блок остаётся читаемым и структура понятна.
- Не дробить секцию на дополнительные визуальные карточки без явной пользы для иерархии.
- Не делать текст малозаметным: вторичный текст может быть слабее основного, но должен оставаться уверенно читаемым.
- Не запускать `bun run typecheck` автоматически после каждого запроса или каждой правки. Запускать проверку типов только по явному запросу пользователя.
- Для `contenteditable` полей не делать реактивную нормализацию, форматирование и принудительную синхронизацию текста во время ввода, если задача требует нативного поведения курсора.

## Tailwind

- Следовать актуальному Tailwind CSS v4.
- Не использовать: `rounded`, `rounded-*`, `shadow`, `shadow-*`, `box-shadow`, `backdrop-blur`, градиенты `bg-gradient-*`, `from-*`, `via-*`, `to-*`.
- Исключение: `rounded-full` только для действительно круглых элементов.
- Состояния `focus`, `hover`, `active`, `selected`, `current` оформлять через `ring`.
- Предпочитать utility-классы вместо кастомного CSS.
- Для theme variables использовать их напрямую в utility-классах Tailwind, например `divide-y divide-(--lab-border)` или `border-(--lab-border)`.
- Не создавать составные helper-классы вроде `lab-divide-soft`, `lab-border-soft` и аналогичные, если то же самое можно записать напрямую через utility-классы и CSS-переменные темы.
- Если в коде встречается `border-(--lab-border)`, удалять его по умолчанию при правках и не добавлять заново без отдельной необходимости.
- Если значение уже выражается стандартной шкалой Tailwind, использовать стандартный utility-класс вместо arbitrary value: например `w-25.5`, а не `w-[102px]`. Arbitrary values использовать только когда точного эквивалента в шкале нет.
- Не использовать `break-words`; использовать `wrap-break-word`.
- Писать `aspect-4/3`, не `aspect-[4/3]`.
- По умолчанию не использовать `border`, `border-*`, `rounded`, `rounded-*` и отдельные фоновые подложки для внутренних дочерних блоков. Если нужен акцент, делать его через типографику, ритм, композицию и точечный цветовой акцент заголовка или маркера.
- Не использовать сочетание `text-inherit` с явным цветом текста, например `text-(--lab-text-primary)`.
- Не использовать `text-inherit`, если цвет текста и так корректно наследуется от родителя.

## Nuxt / Vue / TS

- Использовать `script setup lang="ts"`.
- Не импортировать вручную auto-import сущности.
- Считать auto-import доступным для:
  - `components/`
  - `composables/`
  - `shared/types/*.ts`
  - `shared/utils/*.ts`
  - `app/data`
  - `app/constants`
  - `app/components`
  - `app/composables`
  - `app/utils`
- Имена компонентов должны соответствовать Nuxt auto-import схеме по пути.
- Не использовать `any`.
- Не использовать границы и отступы у потомков общего контейнера
- Упрощать DOM-структуру шаблонов при любой безопасной возможности.
- использовать иконки только из набора Google Materials, стиль round `icon="ic:round-*"`

## Final Check

- Нет запрещённых Tailwind-классов и эффектов.
- Нет лишних контейнеров и промежуточных обёрток.
- `focus/hover/active` оформлены через `ring`.
- Нет ручных импортов auto-import сущностей.
- Структура не стала сложнее без причины.
